# 云谱康官网2.0产品原型架构说明（Codex Vue版）

> 用途：本文件用于给 Codex、ChatGPT、产品经理、UI 设计师和前端开发生成云谱康官网2.0产品原型。  
> 核心原则：页面内容属于云谱康；整体视觉风格参考诺米代谢官网的首页轮播、深色导航和页尾质感；导航交互参考 Sapient 官网的三栏 Mega Menu；栏目主题和业务内容以云谱康已知业务为准。  
> 注意：本文档只保留网站原型架构、页面结构、视觉规范、交互规范和内容需求，不包含建站平台、后台工具或与原型架构无关的信息。

---

## 1. 项目定位

### 1.1 官网2.0核心定位

云谱康官网2.0应定位为：

**以代谢组学和临床质谱技术为核心，连接科研服务、临床检测、健康体检、健康数据管理与临床转化的精准健康科技公司官网。**

官网2.0不是单纯的技术展示站，也不是单一科研服务站，而是一个面向多类客户的综合型企业官网和业务获客入口。

### 1.2 官网2.0要服务的客户

1. **科研客户**：高校、医院科研团队、药企、科研院所，需要代谢组学、蛋白组学、多组学检测与数据分析服务。
2. **医院与临床合作方**：需要临床质谱检测、方法学开发、LDT 项目合作、实验室共建、临床队列合作。
3. **健康体检与健康管理客户**：需要代谢健康评估、慢病风险评估、健康报告、健康数据管理、长期追踪和未病管理服务。
4. **产业与转化合作方**：关注 IVD 试剂盒研发、注册申报、标志物验证、产学研医合作。
5. **政府、园区、投资方、合作机构**：关注公司定位、技术壁垒、研发管线、资质荣誉、学术成果和团队实力。
6. **求职者**：关注公司愿景、团队文化、招聘岗位和加入方式。

### 1.3 官网2.0必须覆盖的云谱康已知业务

- 代谢组学科研服务
- 非靶向代谢组学
- 靶向定量代谢组学
- 胆汁酸、氨基酸、脂肪酸、甾体激素、TMAO 通路、肠道菌群代谢物、色氨酸通路等定量套餐
- 定量蛋白组学
- 多组学联合分析
- 临床质谱检测与临床质谱平台
- 方法学开发与验证
- LDT 项目开发
- IVD 试剂盒研发与后续注册申报
- 临床队列合作与标志物验证
- 健康体检
- 代谢健康评估
- 慢病风险评估
- 健康数据管理
- 报告解读、健康咨询、未病干预、长期追踪与复测
- 技术平台：LC-MS/MS、Orbitrap、SCIEX Triple Quad、样本前处理、标准品与内标体系、数据库、质控体系、数据分析平台
- 学术资源：论文成果、技术文章、行业资讯、公司新闻、下载资料、常见问题
- 公司介绍、实验室环境、资质荣誉、知识产权、合作单位
- 加入我们：招聘岗位、人才理念、团队文化

## 2. 参考网站吸收原则

### 2.1 吸收诺米代谢官网的优点

参考点：

1. **首页大视觉轮播**：采用大面积视觉背景、居中主标题、简洁副标题、轮播指示器，形成第一眼的品牌冲击力。
2. **深色顶部导航**：Logo 左置，导航横向排列，当前栏目高亮，整体有科技感和专业感。
3. **业务并列展示**：科研、临床健康、云平台、媒体、联系我们等入口清晰并列。
4. **深色页尾**：联系方式、公众号二维码、版权信息、备案信息集中在页尾，视觉统一，适合所有页面复用。

云谱康采用方式：

- 首页首屏采用类似全屏 Hero 轮播，但视觉内容改为云谱康自己的“代谢组学、临床质谱、精准健康、数据管理”主题。
- 顶部导航可采用深色或透明深色导航，首屏覆盖在 Banner 上时更有质感。
- 页尾采用深色大页尾，固定出现在每个页面末尾。
- 不照搬诺米代谢的金棕色体系，云谱康主色仍建议使用“深蓝 + 科技蓝 + 白色 + 橙色点缀”。

### 2.2 吸收 Sapient 官网的优点

参考点：

1. **Mega Menu 三栏结构**：左侧显示二级栏目，中间显示当前栏目介绍与关键入口，右侧显示插画/场景图和简介。
2. **导航不仅是链接列表，而是业务说明**：用户打开导航时就能理解每个栏目解决什么问题。
3. **强 CTA 设计**：顶部有明显“Book a Meeting / Contact Us”一类按钮。
4. **从服务到洞察的叙事**：不是只讲技术，而是讲如何从样本、数据、分析到决策。

云谱康采用方式：

- 一级导航打开后使用三栏 Mega Menu：左栏二级菜单，中栏业务解释，右栏插画和一句话介绍。
- 每个 Mega Menu 右侧插画必须与业务相关，例如代谢网络、质谱图、血液样本、健康数据仪表盘、试剂盒、实验室场景等。
- 顶部导航右侧增加主 CTA 按钮，例如“获取方案”或“咨询合作”。
- 页面叙事统一采用“技术能力 → 应用场景 → 服务流程 → 交付结果 → 咨询转化”的结构。

## 5. 顶部导航与 Mega Menu 原型规范

### 5.1 顶部导航布局

参考诺米代谢的横向导航和 Sapient 的清晰 CTA，建议如下：

- 左侧：云谱康 Logo / iPhenome Logo
- 中间：一级导航
- 右侧：语言切换“中 / En” + CTA 按钮“获取方案”或“咨询合作”
- PC 端导航高度：72px 左右
- 首屏可采用透明深色导航；滚动后变为固定深色或白色导航
- 当前栏目使用橙色或科技蓝高亮

### 5.2 Mega Menu 三栏结构

每个带二级栏目的一级导航，悬停或点击后展开 Mega Menu。

推荐结构：

- 左栏：二级栏目列表
- 中栏：当前选中二级栏目的标题、说明、关键入口按钮
- 右栏：业务插画/场景图 + 一句话介绍

交互要求：

- PC 端支持 hover 与 click。
- hover 展开需要 300-500ms 延迟，避免误触。
- 鼠标离开导航区域后延迟关闭。
- 键盘 Tab 可以进入导航项。
- 移动端不使用 Mega Menu，改为抽屉菜单 + 手风琴折叠。

## 6. 首页 Hero 轮播设计规范

### 6.1 整体风格

参考诺米代谢首页轮播的视觉冲击力，但采用云谱康自己的医疗科技风格。

设计方向：

- 大幅全屏或近全屏视觉背景
- 深蓝、科技蓝、白色、橙色点缀
- 背景可使用代谢网络、细胞、质谱峰、血液样本、数据流、分子结构、实验室场景
- 中心或左中对齐主标题
- 文案简洁、专业、有业务指向
- 每张轮播必须有 CTA 按钮
- 轮播页数建议 2-3 张，不要过多

### 6.2 Hero 轮播内容

#### Slide 1：公司定位

主标题：代谢组学与临床质谱驱动的精准健康科技平台  
副标题：覆盖科研服务、临床质谱、健康体检、健康数据管理与临床转化，助力分子表型数据走向精准医学应用。  
按钮：了解云谱康 / 咨询合作

背景建议：深蓝代谢网络 + 质谱峰 + 抽象细胞/血液样本。

#### Slide 2：科研服务

主标题：高覆盖质谱多组学科研服务

## 14. 原型交付检查清单

Codex 或设计工具生成原型后，需要检查：

- 是否保留九个一级导航。
- 是否使用云谱康业务内容，而不是参考网站内容。
- 首页是否有 2-3 张高质量 Hero 轮播。
- 首页是否有四大业务入口。
- Mega Menu 是否是三栏结构，并且右侧有业务插画和说明。
- 每个一级栏目是否有明确二级栏目。
- 每个页面末尾是否有统一深色页尾。
- 页尾是否包含品牌、快速导航、联系方式、二维码、版权、备案占位、隐私政策和网站地图。
- 科研服务是否包含非靶向、靶向、定量蛋白组学、多组学联合分析。
- 靶向套餐详情页是否有科学背景、代谢物列表、谱图、方法学性能、样本送检要求、参考文献和 CTA。
- 临床质谱、健康管理、临床转化是否独立成体系。
- 是否有清晰 CTA：获取方案、咨询合作、下载送样指南、获取检测列表。
- 是否有移动端导航方案。
- 是否使用 Vue 3 + Vite + TypeScript + Tailwind CSS + Vue Router。
- 是否采用 .vue 单文件组件、Composition API 和 script setup。
- 是否将导航、Mega Menu、首页内容、页尾内容尽量放入 data/*.ts，便于后续维护。
- 是否有合规提示和隐私提示。

## 15. Codex 高级原型生成强化要求（复检补充）

为减少 Codex 反复返工，并提高生成高保真产品原型的成功率，建议在实际开发时额外遵守以下强化要求。

### 15.1 Vue 技术边界

- 只能使用 Vue 3、Vite、TypeScript、Tailwind CSS、Vue Router。
- 使用 Vue 3 Composition API、`<script setup lang="ts">` 和 `.vue` 单文件组件。
- 不引入 React、React Router、JSX、Next.js、Redux。
- Pinia 仅在出现复杂全局状态时使用，第一阶段原型通常不需要。
- 咨询表单只做前端模拟提交，不接真实后端。

### 15.2 第一阶段页面完整性

为避免一级导航存在断链，第一阶段至少应生成以下页面：

1. 首页 `Home.vue`
2. 科研服务总页 `ResearchServices.vue`
3. 靶向定量代谢组学页 `TargetedMetabolomics.vue`
4. 临床质谱页 `ClinicalMassSpec.vue`
5. 健康管理页 `HealthManagement.vue`
6. 临床转化页 `ClinicalTranslation.vue`
7. 技术平台页 `TechnologyPlatform.vue`
8. 学术资源页 `AcademicResources.vue`
9. 关于我们页 `About.vue`
10. 加入我们页 `Careers.vue`
11. 404 页面 `NotFound.vue`

首页、科研服务、靶向定量代谢组学、临床质谱、健康管理、临床转化、关于我们应做成较完整高保真页面。技术平台、学术资源、加入我们可以先做成高保真模板页，但必须有 Hero、核心模块、CTA 和页尾，不能空白。

### 15.3 数据文件优先

为便于后续维护，导航、Mega Menu、首页模块、服务内容、页尾内容应尽量放入：

```text
src/data/navigation.ts
src/data/megaMenu.ts
src/data/home.ts
src/data/services.ts
src/data/pages.ts
src/data/footer.ts
```

组件只负责渲染和交互，不要把大量业务文案硬编码在多个页面中。

### 15.4 高级视觉要求

- 使用深蓝、科技蓝、白色、浅蓝灰为主色，橙色作为 CTA 和导航高亮。
- 可使用 CSS 渐变、抽象网格、分子网络、质谱峰图、数据流、玻璃拟态卡片、轻量阴影增强科技感。
- 如无正式图片素材，使用高质量抽象插画占位，不使用廉价医疗素材图。
- 不展示未经授权的医院、高校、合作单位 Logo。
- 首页第一屏必须在 5 秒内说明：云谱康是谁、做什么、服务谁、下一步如何咨询。

### 15.5 交互强化要求

PC 端：

- 顶部导航保留九个一级栏目。
- Mega Menu 必须为三栏结构。
- 支持 hover 和 click 展开。
- 鼠标离开菜单区域后延迟关闭。
- 支持键盘 Tab 访问，Esc 关闭。

移动端：

- 不使用 PC Mega Menu。
- 使用汉堡菜单 + 抽屉菜单 + 手风琴二级导航。
- 点击菜单项后关闭抽屉并跳转。
- 抽屉打开后避免背景滚动。
- CTA 按钮在移动端菜单底部明显展示。

### 15.6 移动端验收宽度

至少检查：

```text
375px
390px
430px
768px
1024px
1440px
```

必须满足：

- 无明显横向滚动条。
- Hero 标题、按钮、轮播指示器不溢出。
- 卡片在手机端单列堆叠。
- 表格在手机端横向滚动或转换为卡片。
- 页尾移动端纵向堆叠。
- 表单输入框和 CTA 适合触控。

### 15.7 构建与验收要求

Codex 完成后必须尝试运行项目可用的检查命令：

```text
npm install，如依赖未安装
npm run type-check，如项目已配置
npm run lint，如项目已配置
npm run build
```

如果某个命令不存在，需要说明原因，不得编造成功。若 `npm run build` 报错，应先修复再交付。
